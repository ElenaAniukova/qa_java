# qa_java

## Описание проекта
В проекте реализованы 4 класса и 1 интерфейс:

- **Animal** — базовый класс для создания животного.
- **Feline** — определяет поведение семейства кошачьих.
- **Cat** — класс, описывающий кота.
- **Lion** — класс, описывающий льва.
- **Predator** — интерфейс, определяющий, является ли животное хищником.

## Цели задания
- Убрать зависимость `Lion` от `Feline` с помощью инъекции зависимостей.
- Написать тесты для классов `Feline`, `Cat` и `Lion`.
- Использовать Mockito для создания моков.
- Реализовать параметризованные тесты.
- Проверить обработку исключений.
- Оценить покрытие Jacoco — для классов `Feline`, `Cat`, `Lion` оно должно быть **100%**.

## Подготовка к тестированию
- Для разработки и запуска тестов использовалась **IntelliJ IDEA**.
- Проект собран с помощью **Maven**.
- Подключены зависимости: **JUnit**, **Mockito**, **Jacoco**.
- В классе `Lion` реализована инъекция зависимости через параметр конструктора `Feline feline`.

## Тестирование

### FelineTest
- Протестированы все четыре метода класса `Feline`.
- Моки не использовались, так как `Feline` не имеет внешних зависимостей.

### CatTest
- Протестированы два метода класса `Cat`.
- Зависимость объявлена как интерфейс `Predator`.
- Для тестирования метода `getFood()` использован мок `Feline`, который реализует `Predator`.

### LionTest
- Использован мок `Feline` для тестирования методов:
    - `getKittens()`
    - `getFood()`
- Протестировано исключение конструктора при передаче некорректного значения пола.

### LionConstructorTest
- Реализован параметризованный тест, проверяющий:
    - корректность работы конструктора;
    - метод `doesHaveMane()` для разных значений пола.
- Использован мок `Feline`.

## Итог
В проекте полностью протестированы и покрыты на **100%** классы `Feline`, `Cat`, `Lion`.  
Использованы моки, параметризация, проверка исключений и принципы инъекции зависимостей.
